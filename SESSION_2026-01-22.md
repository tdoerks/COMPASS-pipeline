## Completed Work

### 1. Fixed Latest 1000 Bacteria Pipeline Bugs

**Bug #1: Organism Diversity Issue**
- **Problem**: Only downloading Campylobacter (all 1000 samples)
- **Root cause**: `metadata_filtering.nf:216` used `.head(1000)` which takes first 1000 rows
- **Fix**: Changed to `combined.sample(n=max_samples, random_state=42)` for random sampling
- **Result**: Now samples proportionally from all organisms
- **Commit**: 10a47ad

**Bug #2: Singularity UID Resolution Error**
- **Problem**: Container error: "user: unknown userid 3600"
- **Root cause**: runOptions commented out in nextflow.config
- **Fix**: Uncommented runOptions with `--no-home --contain` flags
- **History**: This same issue occurred in Oct 2025, was fixed, then got commented out during cleanup
- **Result**: Containers now work on fastscratch
- **Commit**: c629d66

## Important Discoveries

### Singularity UID Bug History
- **October 2025**: Same UID error occurred, was fixed with runOptions
- **December 2025**: Got commented out during "cleanup for main merge" (commit cb2aa7b)
- **Jan 2026**: E. coli runs got "lucky" and worked without the fix
- **Jan 22, 2026**: Latest 1000 bacteria hit the error again, fix re-applied

**Lesson**: Don't comment out production fixes during cleanup!

### Organism Diversity Bug
- `.head()` on concatenated dataframes is order-dependent
- Always use `.sample()` for representative subsets
- This affects ANY pipeline run with multiple organisms and max_samples
