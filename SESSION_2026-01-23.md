→ `2> >(tee ... >&2)` **keeps stderr separate** from stdout

### Tree Format Issues (Newick Colons)

Even when trees are generated correctly (like Kansas), they can't be viewed in iTOL/FigTree because:

**Problem**: Sequence IDs contain colons (`:`) which conflict with Newick format

Example sequence ID from VIBRANT output:
```
kansas_SRR13928117_NODE_6_length_351365_cov:42.366388_fragment_2
```

Newick format uses colons to separate node names from branch lengths:
```
(nodeA:0.5,nodeB:0.3)
```

**Error in tree viewers:**
```
Invalid branch length
Generalized Time-Reversible
. Check if your node IDs contain colons ( : ), and remove or replace them.
```

**Solution**: Clean tree by replacing colons in sequence IDs with underscores, but preserve colons before branch lengths:

```bash
# Extract just the tree (last line, remove FastTree logs)
tail -1 amr_prophage_tree.nwk > tree_only.nwk

# Replace ALL colons with underscores
sed 's/:/_/g' tree_only.nwk > temp.nwk

# Restore colons only before branch lengths (decimal numbers followed by delimiters)
sed -E 's/_([0-9]+\.[0-9]+)([,);])/:\1\2/g' temp.nwk > tree_cleaned.nwk
```

**Result:**
```
Before: kansas_SRR25118668_NODE_3_length_225582_cov:42.366388_fragment_2:0.25788
After:  kansas_SRR25118668_NODE_3_length_225582_cov_42.366388_fragment_2:0.25788
                                                       ↑                        ↑
                                              underscore (ID)             colon (branch length)
```

✅ Kansas AMR-prophage tree successfully cleaned and ready for iTOL!

### Lessons Learned

1. **Stderr redirection matters**: `2>&1 | tee` merges streams incorrectly; use `2> >(tee ... >&2)` for proper separation
2. **Validate alignment files**: Always check that alignment starts with `>`, not log messages
3. **Newick format is strict**: Colons in sequence IDs break all tree viewers
4. **FastTree logs in output**: Tree file should be just the last line (Newick string), not the full log
5. **Process substitution is powerful**: `>(command)` creates a temporary file descriptor for complex redirections
