# COMPASS v1.0.0 Validation Guide

## Overview

This guide explains how to run validation tests on COMPASS v1.0.0 and compare results with the v1.3-dev validation run from 2026-02-09.

## Purpose

Compare COMPASS v1.0.0 (latest release) against v1.3-dev (development version used in February 2026 validation) to:
- Verify core functionality is preserved in v1.0.0
- Identify any differences in tool outputs between versions
- Assess whether v1.0.0 is suitable for production use with your validation dataset

## Validation Data

The validation was run on **163 reference genomes** (E. coli strains) including:
- ETEC reference strains (E925, E1649, E36, E2980, E1441, E1779, E562, E1373)
- FDA ARGOS strains (001, 003, 004, 005, 007, 008)
- Additional reference strains (JJ1886, CFT073, EC958, K12_MG1655, VREC1428, ETEC_TW10722, ETEC_TW11681)

### Previous Validation Results

**v1.3-dev validation (2026-02-09):**
- Location: `/scratch/tylerdoe/COMPASS_Validation_Results_v1.3-dev_2026-02-09/`
- Samples: 163 reference genomes
- Modules tested:
  - QUAST (assembly QC)
  - MLST (strain typing)
  - ABRicate (AMR detection - multiple databases: ncbi, resfinder, argannot, card)
  - MOB-suite (plasmid detection)
  - BUSCO (genome completeness)
  - AMRFinder+ (resistance genes)
  - VIBRANT (prophage detection)

## Setup Instructions

### 1. Prepare the Pipeline

On Beocat, clone the repository to your fastscratch directory:

```bash
cd /fastscratch/tylerdoe/
git clone https://github.com/tdoerks/COMPASS-pipeline.git
cd COMPASS-pipeline
```

### 2. Checkout the Test Branch

```bash
git fetch origin
git checkout test-validation-1.0.0
```

This branch contains:
- COMPASS v1.0.0 codebase
- Validation scripts from the `scratch` branch
- The adapted validation script for v1.0.0

### 3. Prepare Validation Data

You need to create a samplesheet pointing to your validation genomes. The v1.3-dev run used assemblies located in:
- `/fastscratch/tylerdoe/validation_genomes/` (or similar)

**Check your existing samplesheets:**
```bash
ls -lh data/validation/*samplesheet*.csv
```

Available samplesheets:
- `validation_samplesheet_163.csv` - Full 163 genomes (original)
- `validation_samplesheet_fasta.csv` - FASTA mode format
- `etec_samplesheet.csv` - ETEC strains only (8 genomes)

**Verify genome paths in the samplesheet:**
```bash
head data/validation/validation_samplesheet_fasta.csv
```

If paths are incorrect, update them to match your genome locations:
```bash
# Edit the samplesheet
nano data/validation/validation_samplesheet_fasta.csv
```

### 4. Run the Validation

Submit the SLURM job:

```bash
sbatch data/validation/run_compass_validation_v1.0.0.sh
```

**Expected runtime:** 24-48 hours for 163 genomes

**Monitor the job:**
```bash
squeue -u $USER
```

**Check logs:**
```bash
# Find the job ID from squeue or the submission output
tail -f /scratch/tylerdoe/COMPASS_Validation_Results_v1.0.0_<JOBID>/logs/compass_validation_<JOBID>.out
```

## Comparing Results

### v1.3-dev vs v1.0.0 Comparison

**v1.3-dev results location:**
```
/scratch/tylerdoe/COMPASS_Validation_Results_v1.3-dev_2026-02-09/results/
```

**v1.0.0 results location:**
```
/scratch/tylerdoe/COMPASS_Validation_Results_v1.0.0_<JOBID>/results/
```

### Key Metrics to Compare

1. **MLST Results**
   - Compare sequence types (STs) detected
   - Check for consistency in allelic profiles

2. **AMR Detection**
   - Compare gene counts from AMRFinder+
   - Compare ABRicate results across databases
   - Check for differences in resistance profiles

3. **Plasmid Detection**
   - Compare MOB-suite plasmid counts
   - Check incompatibility group assignments

4. **Assembly QC**
   - Compare BUSCO completeness scores
   - Check QUAST assembly statistics

5. **Prophage Detection**
   - Compare VIBRANT prophage counts
   - Check quality scores

### Comparison Commands

```bash
# Set variables
V13_DIR="/scratch/tylerdoe/COMPASS_Validation_Results_v1.3-dev_2026-02-09/results"
V10_DIR="/scratch/tylerdoe/COMPASS_Validation_Results_v1.0.0_<JOBID>/results"

# Compare MLST results
diff <(cut -f1-3 $V13_DIR/mlst/*.tsv | sort) \
     <(cut -f1-3 $V10_DIR/mlst/*.tsv | sort)

# Count AMR genes per sample
echo "=== v1.3-dev AMR counts ==="
wc -l $V13_DIR/amrfinder/*.tsv

echo "=== v1.0.0 AMR counts ==="
wc -l $V10_DIR/amrfinder/*.tsv

# Compare plasmid counts
echo "=== v1.3-dev plasmid counts ==="
grep -c "^" $V13_DIR/mobsuite/*/mobtyper_results.txt

echo "=== v1.0.0 plasmid counts ==="
grep -c "^" $V10_DIR/mobsuite/*/mobtyper_results.txt
```

## Troubleshooting

### Samplesheet Path Issues

If the script fails due to missing genomes:
```bash
# Check if genome files exist
while read line; do
    genome=$(echo $line | cut -d',' -f3)
    if [ ! -f "$genome" ]; then
        echo "MISSING: $genome"
    fi
done < data/validation/validation_samplesheet_fasta.csv
```

### Database Path Issues

Verify database locations in `nextflow.config`:
```bash
grep -E "prophage_db|busco_download" nextflow.config
```

Update paths if needed:
```bash
nano nextflow.config
```

### Memory Issues

If jobs fail due to memory:
```bash
# Edit the SLURM parameters in the script
nano data/validation/run_compass_validation_v1.0.0.sh

# Increase memory: Change --mem=64G to --mem=128G
```
